Deploy Guide

Deploy on a Linux server from scratch in minutes

Install Node.js

Node.js 20 or higher is required.

bash
# Ubuntu / Debian
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# Verify installation
node -v   # Should show v20.x or higher

Get the Language Server

Extract the language_server_linux_x64 binary from Windsurf's Linux release.

bash
# Create directory and place the binary
mkdir -p /opt/windsurf /opt/windsurf/data/db
cp language_server_linux_x64 /opt/windsurf/
chmod +x /opt/windsurf/language_server_linux_x64

Download the Project

bash
git clone https://github.com/guanxiaol/WindsurfAPI.git
cd WindsurfAPI

Configure Environment Variables

Create a .env file and adjust settings as needed.

.env
PORT=3003
API_KEY=                          # Empty = no API key validation
DEFAULT_MODEL=gpt-4o-mini         # Default model
MAX_TOKENS=8192
LOG_LEVEL=info                    # debug / info / warn / error
LS_BINARY_PATH=/opt/windsurf/language_server_linux_x64
LS_PORT=42100
DASHBOARD_PASSWORD=               # Empty = no dashboard password

Start the Service

PM2 is recommended for process management with auto-restart and startup on boot.

bash
# Install PM2
npm install -g pm2

# Start service
pm2 start src/index.js --name windsurf-api

# Enable startup on boot
pm2 save
pm2 startup

Add Windsurf Accounts

After starting the service, add at least one account. The token is obtained by logging into the Windsurf editor.

bash
# Add account via Token
curl -X POST http://localhost:3003/auth/login \
  -H "Content-Type: application/json" \
  -d '{"token": "your-windsurf-token"}'

# Add account via API Key
curl -X POST http://localhost:3003/auth/login \
  -H "Content-Type: application/json" \
  -d '{"api_key": "sk-ws-..."}'

# Batch add multiple accounts
curl -X POST http://localhost:3003/auth/login \
  -H "Content-Type: application/json" \
  -d '{"accounts": [{"token":"t1"},{"token":"t2"}]}'

Verify and Use

bash
# Health check
curl http://localhost:3003/health

# Test chat
curl http://localhost:3003/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o-mini","messages":[{"role":"user","content":"Hello"}]}'

# Open admin dashboard
# Visit http://your-ip:3003/dashboard in your browser
Restart warning: Do not use pm2 restart as it produces zombie processes. Use: pm2 stop && pm2 delete && fuser -k 3003/tcp && pm2 start
