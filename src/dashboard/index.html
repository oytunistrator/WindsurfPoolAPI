Experimental Features

Unstable optimizations; can be disabled at any time if issues arise
Cascade Conversation Reuse
Reuses the same cascade_id across multi-turn conversations, sending only the latest user message to Windsurf so the server maintains the context cache. On a hit, this significantly reduces TTFB and upload size; on a miss or account/LS change it automatically falls back to a new session. Requires the client to maintain full history and append in order (e.g. new-api, OpenWebUI).
Enable Cascade Conversation Reuse
Off by default. Takes effect on the current conversation pool immediately when enabled.
Model Identity Injection
When enabled, injects a system prompt into each request context so the model identifies itself by the requested identity (e.g. when requesting claude-opus-4.6, the model will say it is Claude Opus 4.6, made by Anthropic). Overrides Cascade's built-in Windsurf identity.
Enable Model Identity Injection
On by default. Supports Anthropic / OpenAI / Google / DeepSeek / xAI and other providers.
